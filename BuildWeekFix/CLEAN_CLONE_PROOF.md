**87 total, 82 passed, 5 failed, 0 skipped, and 0 expected failures**. The bundle

## Defects exposed by clean-clone testing

The proof was not declared green on its first attempt:

1. An earlier clean clone failed aggregate static QA because importer validation ran
   before generation of ignored release manifests. The aggregate order was fixed,
   a regression assertion was added, and CI now runs import validation after
   manifest generation. A fresh clone then passed 40/40 from an absent reports
   directory.
2. A later full unit run executed 458 tests and passed 456. Two route-integrity
   tests exposed that published `article:data-project:*` routes rendered in the UI
   but did not round-trip through the string resolver. The resolver now validates
   published dynamic articles through `ContentRepository`, while static guide
   articles remain validated through `GuideContent`. The affected two suites then
   passed 39/39 before the then-authoritative full rerun passed 458/458. Later
   regression coverage expanded the current machine-readable aggregate to 460/460
   (453 Swift Testing cases plus seven XCTest cases).
3. A proposed central lookup compiled with three actor-isolation warnings. It was
   replaced with a main-actor resolver lookup; the authoritative clean build then
   closed with zero warnings.

No failure, warning, skipped test, or workaround run above was reclassified as a
PASS for the authoritative snapshot.
