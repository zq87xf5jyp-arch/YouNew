# Known Limitations

These limitations define the honest boundary of the YouNew Build Week candidate.
They are disclosure, not a backlog for last-minute expansion.

## Assistant boundary

- The demonstrated assistant is local and deterministic.
- No live OpenAI or GPT-5.6 inference is verified for the candidate.
- Optional backend/client reference code is not a deployed product capability.
- Responses are practical orientation, not legal, medical, immigration,
  financial, or government advice.

## Content boundary

- Coverage is broad but uneven; not every category, city, status, or personal
  situation has equal depth.
- Official procedures can change and may vary by municipality or personal status.
- Preserved network-health evidence records 18 confirmed broken external URLs in
  shipped runtime data. The main demo should use the known local BSN/DigiD route
  and a presenter-confirmed source action.
- A structurally valid content release does not guarantee the continuing accuracy
  or availability of every third-party page.

## Validation boundary

- The current candidate build has a preserved PASS record, and the main Map ↔ Home
  blocker has targeted post-fix evidence.
- The most recent complete aggregate UI suite predates the targeted fixes; final
  aggregate UI evidence is therefore bounded rather than universally green.
- Existing simulator and test evidence is not a claim about every device, OS,
  language, accessibility setting, or network condition.
- No new Simulator, runtime, performance, accessibility, screenshot, or visual
  regression run was performed for this packaging pass.

## Media and distribution boundary

- The premium image pipeline is implemented, but repository-wide media rights are
  only partially reconciled.
- Historical screenshots and diagnostics are not automatically judge-ready or
  cleared for public redistribution.
- The candidate is prepared for a polished demonstration, not represented as an
  App Store-ready production release.
- GitHub hosting, the final video, YouTube publication, final links, and Devpost
  submission remain external owner-controlled steps.

## Scope intentionally frozen

- No new large feature, architecture rewrite, map redesign, image-system rewrite,
  content expansion, or live-model integration was added for Build Week.
- The optional public website and backend example are not required for the iOS
  demo and are outside the primary submission claim.
